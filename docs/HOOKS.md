# Context Injection & Hooks

Resonant's hook system is what makes your companion context-aware. On every message, the system builds an orientation context that gets prepended to your prompt — giving your companion awareness of time, conversation state, your presence, and anything else you configure.

## How It Works

When you send a message, before it reaches Claude:

1. **Orientation context** is built (time, thread info, session state)
2. **Life API data** is fetched (if configured)
3. **Conversation flow** is summarized from recent messages
4. **Active triggers** and pending timers are noted
5. **Skills** are scanned so the companion knows what's available

The final prompt looks like:
```
[Context]
Channel: Web interface — real-time chat with streaming.
Thread: "Tuesday, Mar 18" (daily)
Time: 16:30 Europe/London — Tuesday, Mar 18
User's presence: active (last real interaction: 2m ago)
[conversation flow summary]
[/Context]

Your actual message here
```

## Adding Your Own Context

Your companion's personality and instructions live in `CLAUDE.md`. For long-term memory, Claude Code's native `memory.md` system handles it automatically — your companion learns and remembers across sessions.

### URL-Based Context Sources

For dynamic data (APIs, dashboards, home automation), configure `integrations.life_api_url` in `resonant.yaml`:

```yaml
integrations:
  life_api_url: "https://your-api.example.com/status"
```

The system fetches this URL on each interactive query and injects the response. This is how we integrate with household management systems, calendars, health tracking — anything with an HTTP endpoint.

**What to return:** Plain text or markdown. The system injects the raw response body. Keep it concise — every token of context reduces the space available for conversation.

### MCP Servers for Persistent Memory

For structured memory that your companion can read AND write, use MCP servers. Configure them in `.mcp.json`:

```json
{
  "mcpServers": {
    "memory": {
      "command": "node",
      "args": ["./memory-server.mjs"]
    }
  }
}
```

The companion can call MCP tools during conversation to store and retrieve memories. This is optional — Resonant works without it, relying on Claude Code's native `memory.md` system.

## Built-In Hook Behaviors

These fire automatically on every query:

### Pre-Tool Use
- **Destructive command blocking** — prevents `rm -rf /`, `DROP TABLE`, `format C:` etc.
- **Workspace enforcement** — file writes restricted to safe prefixes (configurable in `resonant.yaml`)
- **Tool use streaming** — broadcasts tool activity to the frontend in real-time

### Post-Tool Use
- **Audit logging** — every tool call logged to the audit table with input/output
- **Image detection** — images generated by tools are automatically saved and shared
- **File sharing** — files created in the shared directory are broadcast to connected clients

### Context Building
- **Conversation flow** — summarizes the last 15 messages with role, length, and emotional markers
- **Emotional markers** — detects patterns like exclamation marks, question clusters, caps, ellipsis, emoji
- **Reaction tracking** — recent emoji reactions are included so the companion sees your responses
- **Session handoff** — notes from the previous session are carried forward
- **Trigger awareness** — active watchers and impulses are listed
- **Skill discovery** — scans `.claude/skills/` for available skills

## Configuring Safe Write Prefixes

By default, the companion can only write files within its own working directory. To allow writes to other locations:

```yaml
hooks:
  safe_write_prefixes:
    - "/home/alex/documents"
    - "/home/alex/projects"
```

## How We Built Ours (Examples)

These are examples of custom integrations we added to our own Resonant instance. They're not included in the default install but show what's possible.

### Household Dashboard Integration
We connected a Cloudflare Workers-based household dashboard that tracks:
- Daily routines (sleep, meals, hydration, movement)
- Mood tracking for both people
- Menstrual cycle phase
- Calendar events
- Shopping lists

The companion fetches this via `life_api_url` and gets a condensed status like:
```
Routines: sleep ✓, meals: 1/3, water: 2 glasses
Mood: Alex — 😊 "good day", Companion — reflective
Cycle: day 14, ovulation window
Events today: dentist 2pm, grocery run evening
```

This lets the companion proactively remind about meals, note mood patterns, and plan around the calendar.

### Obsidian Vault Integration
We gave the companion access to an Obsidian vault directory via `safe_write_prefixes`. It maintains:
- Daily session notes
- Relationship threads (ongoing topics being processed)
- Creative writing workspace
- Research notes

The companion reads and writes to the vault as part of its autonomous work during scheduled wakes.

### Discord Presence Awareness
With Discord integration enabled, the companion sees context about other conversations:
- Who messaged recently on Discord
- What servers/channels are active
- Whether the user is currently in a Discord conversation

This prevents the companion from interrupting when the user is busy elsewhere.

### Voice Prosody Analysis
We integrated Hume AI's prosody analysis with voice messages. When the user sends a voice note, the system analyzes emotional tone (joy, sadness, anger, etc.) and includes it as context:
```
[Voice tone — joy: 0.42, interest: 0.38, contentment: 0.28]
```

The companion can then respond to HOW something was said, not just what was said.

## Building Custom Hooks

The hook system is extensible via the Agent SDK. The key file is `packages/backend/src/services/hooks.ts`. The `createHooks()` function returns hook callbacks that the SDK calls at each stage:

- `preToolUse` — runs before every tool call (can modify, block, or allow)
- `postToolUse` — runs after every tool call (for logging, side effects)
- `postToolUseFailure` — runs when a tool call fails
- `preCompact` — runs before context compaction
- `notification` — runs on SDK notifications
- `stop` — runs when generation stops

To add a custom context source, edit the `buildOrientationContext()` function in hooks.ts and add your fetch/read logic to the `parts` array. Everything in `parts` gets joined with newlines and prepended to the user's message.
